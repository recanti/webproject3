Notes

One of the first things I did was fix the issues with the first iteration of the site. Everything validates and site position is made evident through a breadcrumbs navigation in the upper left. The header and footer use an ID tag for styling and I use and embedded Oswald font for the splash page title.

I also tried to fix some of the things I wanted to fix with the last iteration of the site. I feel like my Sass files are a lot more organized and focused than they were in the last iteration of the site. It is also more mobile-friendly. Large images resize with the page and in the projects section, the buttons resize to fill the width of the screen.

My Grunt workflow remained relatively the same, with the exception of a few updates to account for a new directory structure. I like the workflow a lot better now and think that the use of Grunt and Sass qualifies as an above and beyond contribution to the project.

There are some things that I am not quite satisfied with. I think that the splash page is a bit boring and could use an image. The same is true of the 404 page. I also think that I've reached the limit of what I can automate with the Grunt and Sass. Managing headers and footers between different directories became a hassle. I think that a Content Management System like Jekyll would work well for this site, but I didn't have time to implement it.

Once again, the theme was a modified version of this palette. Technologies were Grunt and Sass, as well as some instances of JQuery in the about page, as well as the Formmail php script. You can view all of the code on my github.
